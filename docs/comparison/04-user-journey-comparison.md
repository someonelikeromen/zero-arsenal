## 1. 目标用户分层

| 用户类型 | 描述 | zero-arsenal 匹配度 |
|----------|------|---------------------|
| **休闲新手** | 想"直接玩"，不想配置，期待游戏级引导 | 2/5（无引导，TRPG术语未解释）|
| **TRPG玩家** | 了解规则书，想自定义骰子/角色/世界 | 4/5（骰子/规则齐备，但文档学习曲线高）|
| **AI写作用户** | 需要AI辅助长篇叙事，关注文风/记忆 | 3.5/5（文风数据有但无UI；记忆系统偏技术）|
| **创意协作者** | 有GM+玩家角色概念，多人或GM引导 | 3/5（play/plan/review模式支持，但无多人实时功能）|
| **开发/二创者** | 想开发世界扩展、自定义Agent | 4/5（扩展系统完整，文档偏设计而非上手）|

## 2. 核心用户旅程：「首次体验」

### 2.1 zero-arsenal（当前状态）

```
访问 Hub
  │
  ├─ 没有任何引导
  ├─ 看到七个Tab，不知道从哪里开始
  │
  ├─ 自行点击 Settings → 配置LLM API Key（无提示需要这步）
  │
  ├─ 点击 Worlds → 「暂无世界模板」
  │    └─ 点击「创建世界」→ 填写表单（无字段说明）
  │
  ├─ 点击 Characters → 「暂无人物模板」
  │    └─ 点击「创建角色」→ 5步向导（这里体验较好）
  │
  ├─ 点击 Sessions → 「创建会话」（无向导，单页表单）
  │    └─ 选world_plugin/world/character → 创建
  │
  └─ 进入 Session → 看到空白聊天区 → 不知道输什么
       └─ 输入文字 → 等待LLM（无进度显示）→ 看到叙事输出
```

**主要摩擦点**：5个以上决策点，无一有引导文案；API未配置时直到会话第一次调用才报错（不是前置提示）

### 2.2 KoboldCpp（对标最佳）

```
打开应用
  ↓
弹出「选择UI风格」（4张预览卡）← 第一个交互建立归属感
  ↓
「选择使用模式」（4个选项，每个有一句话描述）← 5秒决策
  ↓
「Quick Start - 选择场景」（内置场景库）← 立刻有内容
  ↓
直接输入，立刻可玩
```

**关键差异**：KoboldCpp的第一个交互是个性化选择（主题），而不是"填空"。

### 2.3 SillyTavern（最低摩擦）

```
打开 → 欢迎页 → 点「Temporary Chat」← 2步，立刻可输入
（不需要角色、不需要配置，任何时候都能开始临时对话）
```

### 2.4 Chub.ai（社区内容消除空白）

```
注册（3步）→ 看到热门角色库 ← 立刻有东西看
→ 点击任意角色→「Chat」← 2步开始对话
（社区内容天然消除了空白状态问题）
```

## 3. 核心用户旅程：「创建新内容」

### 3.1 创建角色对比

| 项目 | 步骤数 | 最小必填 | AI辅助 | 预设模板 |
|------|--------|----------|--------|---------|
| **zero-arsenal** | 5步（最完整）| 名称+类型 | ✓ SSE流式预览 | ✗ |
| **SillyTavern** | 1步（单页）| 名称唯一必填 | ✗ | ✗（CharCardStudio扩展）|
| **Chub.ai** | 1步（单页）| 名称+描述+Initial Message | ✗ | ✗ |

## 4. 用户分层需求满足度

### 4.1 新手路径（Goal: 10分钟内看到第一条叙事输出）

| 项目 | 实现度 | 主要策略 |
|------|--------|---------|
| **KoboldCpp** | 4.5/5 | Quick Start场景库 + 主题选择 |
| **SillyTavern** | 4.5/5 | Temporary Chat，零配置可用 |
| **Chub.ai** | 4.0/5 | 社区内容填充，1步开始对话 |
| **MoRanJiangHu** | 3.5/5 | 五步向导，有默认值+预设 |
| **zero-arsenal** | **1.5/5** | 无引导，需完成5+步配置 |

**提升zero-arsenal新手路径的最快方案**：
1. 首次进入Hub时检测`Settings.llmKey`是否配置，未配置则显示顶部Banner「请先配置AI模型」并附链接
2. Sessions Tab添加「快速开始」入口，使用内置Demo场景（一键加载预设世界+角色）
3. 会话创建完成后自动触发第一条叙事（使用角色的first_message或世界opening_scene字段）

### 4.2 熟练用户路径（Goal: 精细控制 + 高效操作）

| 项目 | 实现度 | 主要功能 |
|------|--------|---------|
| **SillyTavern** | 5/5 | Mad Lab Mode + 完整ST-Script + 自定义CSS |
| **NovelAI** | 4.5/5 | Config深度参数 + Advanced Lorebook条件 + 上下文查看器 |
| **zero-arsenal** | **3.5/5** | OOC斜杠命令 + LlmRoutesTab + PromptManager + API /docs |
| **MoRanJiangHu** | 3.5/5 | 分角色绑定模型 + 故障转移 + 20+ Settings Tab |

**zero-arsenal老手功能的主要缺口**：
- 无取消生成（对高频用户是重要控制点）
- 无当前token计数/上下文窗口可视化
- PromptManager缺版本对比和场景标签

## 5. 关键情感旅程节点

```
[发现] → [注册/安装] → [首次配置] → [首次体验] → [深度使用] → [分享/扩展]
```

| 阶段 | zero-arsenal 问题 | 建议解法 |
|------|-----------------|---------|
| **首次配置** | 无检测LLM配置的提示；API未配置时等到第一次生成才报错 | 首次进Hub检测+顶部Banner |
| **首次体验** | 空白会话，用户不知道输什么 | First Message自动触发（NPC先说话）|
| **第二次使用** | Hub会话历史在「存档」Tab，不在「会话」Tab | 合并会话Tab：列表+创建+历史 |
| **深度使用** | 38个文风无法在UI选择 | 文风选择器面板 |
| **遇到问题** | 无Toast反馈，操作成功/失败不透明 | 接线addNotification + Toast UI |
