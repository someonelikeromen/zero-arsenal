# 开发须知

欢迎参与 **零度武库（Zero Arsenal）** 开发。本文是协作者的中文入口；详细规范见 [CONTRIBUTING.md](./CONTRIBUTING.md)，架构与降级行为见 [ARCHITECTURE.md](./ARCHITECTURE.md)，当前进度见 [PROGRESS.md](./PROGRESS.md)。

## 1. 五分钟跑起来

### 环境

- Python **3.11+**
- Node **18+**
- 可选：Redis（多实例 / 多 worker 部署时设置 `REDIS_URL`）

### 后端

```bash
cd zero-arsenal
pip install -e "backend/[dev]"
cp .env.example .env   # 至少填一个 LLM API Key
python -m backend.main
```

API 文档：<http://localhost:8000/docs>

### 前端

```bash
cd frontend
npm install
npm run dev
```

界面：<http://localhost:5173>

### 提交前自测

```bash
pytest backend/ -m "not stub" -q
cd frontend && npm run typecheck && npm run lint && npm run build
```

## 3. 开发铁律

1. **安全路径 fail-closed**：鉴权、verdict、权限解析异常时拒绝或 `ask`，不要回落到 `allow`。
2. **扩展动态加载**：`plugin.py` / `tools.py` / `hooks.py` 无包上下文；`hooks.py` 钩子类须能**无参实例化**。
3. **禁止** `except Exception: pass`；至少打日志，安全相关必须拒绝。
4. **不要提交** `.env`、`.db`、`node_modules/`、`frontend/dist/`。
5. **实现与 design 冲突**：先开 Issue 讨论，再以代码 + 设计文档同步为准（见 `docs/design/`）。

## 4. 常见任务速查

| 任务 | 步骤摘要 |
|------|----------|
| 新增 Agent | 新建 `agents/<name>_agent.py` → `graph.py` 注册 → `permission.py` 白名单 → `profiles/*.yaml` |
| 新增世界扩展 | 复制 `extensions/_template` → 填 `manifest.json` → 实现 `plugin.py` / `tools.py` / `hooks.py` |
| 新增 API | `api/routers/` 加路由，必要时更新 `docs/design/11-api-design.md` |
| 改表结构 | 改 `schema.py` 的 CREATE + `MIGRATION_PATCHES_SQL`，本地跑 `init_db` 验证 |
| 新增内置工具 | `tools/builtin_tools.py` 或扩展 `tools.py` 的 `TOOLS` 列表 |

扩展骨架说明：[backend/extensions/_template/README.md](../backend/extensions/_template/README.md)

## 5. Git 与 PR

- 分支：`feat/xxx`、`fix/xxx`、`docs/xxx`
- 提交信息：`type: 简述`（如 `fix: chronicler cursor`、`docs: update PROGRESS`）
- PR 需写：动机、改动范围、测试命令、UI 截图（如有）
